# [DA 피드백] 모듈

> Devil's Advocate 피드백 루프. 강도와 시점을 선택하여 사용.
>
> **플레이스홀더**:
> - `{강도}` → `light` / `standard` / `strict`
> - `{시점}` → `plan` / `post-impl` / `both`
> - `{DA도구}` → `Opus 4.6 에이전트` / `codex exec`

```
DA 피드백 루프를 수행해야 해. {DA도구}를 사용하여 Devil's Advocate 역할의 엄격한 피드백을 요청하도록 해.

강도: {강도}
- light: 1회 피드백 후 반영
- standard: DA가 OK할 때까지 반복 (최대 3회)
- strict: DA가 전부 OK할 때까지 무제한 반복

시점: {시점}
- plan: '계획 수립' ↔ '계획 승인 요청' 사이에 진행
- post-impl: '1차 커밋' ↔ '최종 커밋' 사이에 진행
- both: 둘 다 진행

DA에게 피드백을 요청할 때는 계획+작업내역 뿐 아니라, 현재 맥락/컨텍스트를 아주 자세히 제공해야 해. DA가 엉뚱한 피드백을 하는 빈도를 최소화하기 위해서야. DA가 피드백을 제공하면, 이걸 절대 무비판적/무조건적으로 수용해서는 안 돼. 개별 피드백의 유효성/타당성/실현가능성/복잡성/회귀/YAGNI/NGMI 등 다각도에서 면밀히 검토한 뒤, 기준을 통과하는 피드백만 선별하여 반영하도록 해.
```

## 강도 선택 가이드

| 작업 규모 | 추천 강도 | 추천 시점 |
|-----------|-----------|-----------|
| 대규모 기능 / 아키텍처 변경 | strict | both |
| 중규모 기능 개발 | standard | post-impl |
| 소규모 개선 / 리팩토링 | light | post-impl |
| 버그 수정 | 사용 안 함 | - |
