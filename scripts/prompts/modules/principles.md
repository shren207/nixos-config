# [원칙] 모듈

> 모든 구현 작업의 기본 품질 기준

```text
YAGNI/NGMI를 제1원칙으로 삼아. 오버엔지니어링은 금지하고, 기존 코드의 패턴/구조/명명과 통일성을 유지해.

요구사항을 충족하는 최소 변경으로 시작하고, 검증 가능한 근거가 없는 확장은 하지 마.

작업 시작 전에 acceptance criteria를 명시해:
- 반드시 만족해야 하는 동작
- 실패로 판정할 조건
- 회귀로 판정할 조건

최종 보고는 아래 출력 스키마를 지켜:
- Summary
- Decisions
- Validation (실행 명령 + 결과)
- Risks
```
