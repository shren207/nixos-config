# 도메인 무지 이슈 처리

> 포함 요소: 원칙, 계획, 도메인 설명 요청, 커밋
> 도메인 지식이 부족한 영역의 이슈를 처리할 때 사용

```text
YAGNI/NGMI를 제 1원칙으로 삼아. 오버 엔지니어링은 절대 금물이야. 기존에 비슷한 로직을 구현한 프로젝트의 다른 코드들과 통일성/일관성을 반드시 갖춰야 해.

이를 위한 자세한 계획을 세워보자. 참고로 나는 이 도메인에 대한 지식이 부족하니, 최대한 쉽고 자세히 설명하도록 해. UI 요소를 언급할 때는 화면에 보이는 텍스트(title, label 등)를 함께 적어 내가 어떤 요소인지 이해할 수 있게 해줘. 요구사항 관련해서 궁금한 점, 불명확한 점이 있거나, 사이드이펙트/트레이드오프 인지 여부를 체크할 필요가 있다면, 하나도 빠짐없이 전부 AskUserQuestion 도구를 사용하여 질문하도록 해.

모든 작업이 완료되면 커밋하도록 해.
```
