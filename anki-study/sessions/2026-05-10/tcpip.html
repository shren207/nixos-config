⚠️ 책 표현 정정 #1: "네트워크 액세스 ↔ 데이터링크" 1:1 매핑
책의 매핑은 부정확하다. 정확히는 TCP/IP의 네트워크 액세스(또는 Link) 계층은 OSI의 데이터링크 + 물리 계층 둘 다를 포함한다.
  • RFC 1122: "Link Layer" — 호스트가 직접 통신 매체에 신호를 올려보내는 일까지 포함
  • 책의 "물리 계층은 별도 견해" ⚠️ 박스가 이 부분을 살짝 인정한 것
  • 면접에서 묻는다면: "TCP/IP의 네트워크 액세스 ↔ OSI의 L1+L2"가 정답
⚠️ 책 표현 정정 #2: "TCP/IP 응용 = 응용+표현+세션" 흡수
정확하지만 단순화. 실제로 TCP/IP에서 표현/세션 기능은 응용 계층 안에 통합되어 있거나, 별도 라이브러리/프로토콜로 분리되어 있다.
  • 표현 계층 기능 (암호화/인코딩) → TLS (TCP 위에 얹는 계층)
  • 세션 계층 기능 (연결 관리) → 응용 프로토콜이 직접 (HTTP/2 stream, WebSocket)
→ "흡수"라기보다 "재배치"가 더 정확.
🚫 YAGNI: 외울 필요 없음
학습 부담을 늘리는 책의 부분들 (외워도 면접/실무 가치 낮음):
  • "두 모델의 직접적이고 엄밀한 비교는 어려울 수 있다" — 메타 진술
  • "TCP/IP에 물리 계층을 추가한 확장 모델로 설명한다" — 한국 교재의 절충, 본질 아님
  • OSI 표현 계층의 ASN.1, 세션 계층의 RPC 같은 옛 프로토콜
→ 시간 아껴서 위의 매핑 + 프로토콜 표에 집중.
🔍 깊이 보충: 패킷이 계층을 내려갈 때 무슨 일이?
[응용]   "GET /tcpip.html"           ← HTTP 헤더 + 본문
[전송]   [TCP 헤더][...]             ← 포트, 시퀀스 번호 추가
[인터넷] [IP 헤더][TCP 헤더][...]    ← 출발/도착 IP 추가
[액세스] [Eth 헤더][IP 헤더][...]    ← 출발/도착 MAC 추가
         → 전기/전파 신호로 매체에 송출
이를 encapsulation(캡슐화)이라고 부르며, 받는 쪽은 역순으로 헤더를 벗긴다 (decapsulation). 이게 OSI/TCP/IP 모델이 실제로 작동하는 메커니즘.
🔍 깊이 보충: 면접에서 "5계층 모델"이라 하면?
Tanenbaum 같은 일부 교재는 "TCP/IP 5계층"을 쓰기도 한다 — 네트워크 액세스를 데이터링크 / 물리로 분리한 절충판.
  • 4계층 (RFC 표준): 응용 / 전송 / 인터넷 / 네트워크 액세스
  • 5계층 (절충): 응용 / 전송 / 네트워크 / 데이터링크 / 물리
어느 쪽을 쓰든 TCP/IP는 응용 계층에서 표현·세션을 흡수한다는 사실은 동일.
