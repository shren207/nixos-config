🔀 Stage 4 / 5 · 인터페이스와 NIC의 관계

"인터페이스 = NIC"가 아닌 이유 (총 30pt)
이게 이 카드의 제일 까다로운 부분이야. 인터페이스와 NIC는 1:1이 아닐 수 있다. 세 가지 케이스 각각 구체적인 예시를 적어봐.
① 1개의 NIC → 여러 인터페이스
② 여러 NIC → 1개의 인터페이스
③ NIC 없는 인터페이스 (=물리 하드웨어 없이 존재)

📌 NIC 용어 정리
NIC = Network Interface Controller
별칭 (모두 자주 사용 — 외워 둘 것):
  • 네트워크 카드 / 어댑터
  • 이더넷 카드 / 어댑터
  • LAN 카드 / 어댑터
지칭 범위
카드 / 어댑터NIC 전체
포트NIC의 물리 연결부만
예) "M1 맥북엔 이더넷 포트가 없어" — NIC는 있지만 물리 포트가 없다는 뜻.
📌 NIC가 하는 일 (책 기준)
  1. 통신 매체 신호 ↔ 호스트 프레임 변환
  2. MAC 주소 기반으로 잘못 전송된 패킷 검증 (FCS; Frame Check Sequence)
📌 인터페이스 ≠ NIC, 3가지 케이스
관계예시
1 NIC → N 인터페이스VLAN 서브인터페이스: eth0.100, eth0.200
N NIC → 1 인터페이스본딩/티밍: bond0 (Linux), LACP (link aggregation)
NIC 없는 인터페이스루프백 lo0, 터널 tun0/tap0, 브리지 bridge0 (Docker의 docker0도 동일)
📌 macOS의 en0
en0의 en은 본래 ethernet 약자.
예전 Mac: en0 = 유선 LAN, en1 = Wi-Fi.
오늘날 (애플 실리콘 맥): en0 = Wi-Fi 인 경우가 많음 (이더넷 포트가 없으니까). → 직관과 어긋남. 시스템에서 networksetup -listallhardwareports로 실제 매핑 확인.
⚠️ 책 표현 보완: "네트워크 인터페이스 = NIC"는 일상 용법
엄밀히는 다른 개념이지만, 일상에서는 거의 같이 쓰인다는 점도 받아들여야 해. 예: ip link show로 보이는 eth0은 OS 측 인터페이스이지만, 실무에서 "이더넷 NIC"라고 부르기도 함. 중요한 건 "VLAN/본딩/루프백 시나리오에서는 두 개념이 갈라진다"는 점.
🔍 깊이 보충 #1: 현대 NIC는 "단순 변환 장치"가 아니다
책의 NIC 역할 (신호 ↔ 프레임 변환, FCS 검증)은 1990년대 NIC 수준. 현대 NIC는 CPU 부담을 덜기 위해 많은 처리를 직접 한다 (= 오프로드):
  • TSO/GSO (TCP Segmentation Offload): 큰 데이터를 NIC가 패킷 단위로 분할
  • LRO/GRO (Large/Generic Receive Offload): 작은 패킷을 NIC가 합쳐서 한 번에 OS에 전달
  • RSS (Receive Side Scaling): 멀티큐로 여러 CPU 코어에 패킷 분산
  • Checksum offload: TCP/IP 체크섬을 NIC가 직접 계산
이 덕에 10/40/100Gbps 네트워크에서도 CPU가 다 떠안지 않는다.
🔍 깊이 보충 #2: 인터페이스 ↔ 메모리 데이터 흐름
NIC는 단순히 신호를 변환하는 게 아니라, DMA로 호스트 메모리에 직접 패킷을 쓴다 (write). 
패킷 도착 → NIC가 DMA로 ring buffer에 저장 → NIC가 IRQ 발생
       → OS의 IRQ 핸들러가 NAPI poll 시작 → softirq에서 패킷 처리
이 흐름은 너의 다른 카드(DMA 입출력)와 직접 연결돼. 나중에 같이 묶으면 이해가 깊어져.
🔍 깊이 보충 #3: 가상 인터페이스의 위력
tun0/tap0 같은 NIC 없는 인터페이스는 VPN/컨테이너의 핵심.
  • 너의 Tailscale이 만든 tailscale0 인터페이스가 정확히 tun 타입
  • Docker의 docker0는 bridge 인터페이스로 컨테이너들을 묶음
  • 이런 가상 인터페이스에 IP 할당 + 라우팅 테이블 조작으로 "물리 NIC 없이도 별도 네트워크"를 만든다
→ ip link show로 직접 확인 가능. 지금 너의 minipc에서도 tailscale0이 살아있어.
💡 한 줄 요약
인터페이스는 OS가 만드는 논리 객체(라우팅·방화벽 기준), NIC는 그것의 물리적 구현체일 뿐. 그래서 1:N, N:1, 0:N 모두 가능하다.
