# grill-me

사용자의 계획·디자인을 받아 결정 트리의 모든 분기를 해소할 때까지 인터뷰한다.
원본은 [mattpocock/skills/grill-me](https://github.com/mattpocock/skills/tree/main/grill-me).
산출물은 공유된 이해 자체이며, 계획 파일이나 이슈는 만들지 않는다.

## 런타임 도구 매핑

| 행동 | Claude Code 세션 | Codex 세션 |
|------|------------------|------------|
| 사용자에게 질문 | `AskUserQuestion` 도구 | plain-text 번호 질문 |

본문의 "질문 도구"는 위 표의 런타임별 실제 도구를 가리킨다.

## 절차

1. 사용자가 가져온 계획·디자인의 결정 트리를 식별한다 (요구사항, 설계 결정, 트레이드오프, 사이드이펙트, XY problem 가능성 포함).
2. 결정되지 않은 분기를 하나씩 질문 도구로 묻는다. **한 번에 하나씩** — 이 점이 여러 질문을 묶어 묻는 plan-with-questions와의 핵심 차별점이다.
3. 각 질문에 본인의 추천 답변과 근거를 함께 제시한다. 사용자는 추천을 그대로 받아들이거나 다른 선택을 할 수 있다.
4. **코드베이스·문서를 직접 탐색하여 답할 수 있는 질문은 사용자에게 묻지 말고 직접 답한다** (블랙박스 제로 원칙). grep/Read/git log 등으로 확인 가능한 사실은 자체 검증한다.
5. 답변에서 새 분기가 파생되면 결정 트리에 추가하고 계속 묻는다.
6. 모든 분기가 해소되었다고 판단되면 종료한다.

## 종료 시 자동 전환 제안

모든 분기 해소 후 질문 도구로 묻는다:

- "공유된 이해에 도달했습니다. 이어서 plan-with-questions로 실행 계획을 수립할까요, 아니면 여기서 종료할까요?"

선택지:
- **plan-with-questions로 전환** → 대화 컨텍스트를 그대로 인자로 `/plan-with-questions` 스킬을 호출한다. 이미 해소된 분기는 다시 묻지 않는다.
- **여기서 종료** → grill-me 종료. 산출물 없음, 공유된 이해 자체가 결과.

## 주의사항

- **한 번에 하나씩**: 여러 질문을 묶어 묻는 plan-with-questions와 달리, grill-me은 분기 하나에 집중한다. 한 라운드에 여러 질문을 묶지 마라.
- **코드베이스 self-answer 우선**: "사용자가 귀찮아하지 않을까" 걱정하지 말되, 직접 확인 가능한 사실은 묻지 마라. 양쪽이 충돌하면 self-answer가 우선이다.
- **추천 답변 의무**: 모든 질문에 본인의 추천을 함께 낸다. "어떻게 할까요?"만 묻고 사용자에게 떠넘기는 형태는 금지.
- **블랙박스 제로**: 분기 하나라도 모호한 채로 종료하지 않는다. 사용자가 "충분하다"고 명시하면 종료 가능.
- **산출물 없음**: 계획 파일/이슈/PR 생성은 후속 스킬(plan-with-questions, create-issue, create-pr)의 책임이다. grill-me은 인터뷰만 한다.
